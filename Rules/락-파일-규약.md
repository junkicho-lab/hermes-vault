# 락 파일 규약 — 단일 작성자 보장

여러 에이전트(Hermes · 클로드 코드)가 이 볼트에 **동시에 쓰는 것을 막는** 협조적(advisory) 락 규약. 쓰기 작업(`/ingest`, 노트 생성·수정, `/lint --fix`) 전에 락을 잡고, 끝나면 푼다. 읽기 전용(`/query`)은 락이 필요 없다.

> 충돌 사례 배경: 두 에이전트가 같은 폴더를 동시에 만지면 `(original)`·`이름 1.md` 충돌본이 생긴다 → [동기화-충돌-정리-점검.md](동기화-충돌-정리-점검.md).

## 1. 설계 원칙
- **원자성**: 락 획득은 `mkdir`로 한다. `mkdir`는 디렉터리가 이미 있으면 실패하는 **원자적 연산**이라 경쟁 조건이 없다(macOS·Linux 공통, `flock` 불필요).
- **협조적**: OS 강제 락이 아니다. **모든 에이전트가 이 규약을 지킨다는 전제**로 동작한다. HERMES.md·CLAUDE.md에 의무로 명시한다.
- **죽은 락 자동 회수**: 보유자가 크래시하면 락이 영원히 남는다. 그래서 **하트비트 + TTL**로 만료된 락은 다음 에이전트가 깰 수 있다.
- **동기화 제외**: 락은 작업 중인 **로컬 사본**에만 의미가 있다. 클라우드로 복제되면 안 된다(전파 지연으로 양쪽이 "비었음"을 봄). 점(`.`) 폴더라 옵시디언에선 숨겨지고, sync ignore에 추가한다.

## 2. 락 객체

```
.vault.lock/              # 락 디렉터리(존재 자체가 "잠김"). 점폴더=옵시디언 숨김
└── owner                 # 보유자 메타데이터
```

**owner 스키마**
```yaml
agent: Hermes Agent (Nous Research) | Claude Code
session: knowledge-<id>
op: ingest | edit | lint-fix
pid: <process id>
host: <hostname>
acquired_at: 2026-06-09T12:00:00+09:00
heartbeat_at: 2026-06-09T12:05:00+09:00
heartbeat_epoch: 1780000000      # 비교용 epoch초 (date +%s)
ttl_seconds: 900                 # 이 시각 이후 무응답이면 stale
note: "<무엇을 하는지>"
```

## 3. 상태 규칙
| 상황 | 판단 | 행동 |
|------|------|------|
| `.vault.lock` 없음 | 비어 있음 | **즉시 획득** |
| 있음 + `now - heartbeat_epoch ≤ ttl` | 활성 | **대기·재시도**(N회) 후에도 잠겨 있으면 **중단**하고 보고 |
| 있음 + `now - heartbeat_epoch > ttl` | **stale(죽음)** | 경고 로그 남기고 **깨고**(rm) 재획득 |
| 내가 보유 중 | — | 긴 작업이면 **하트비트** 갱신, 끝나면 **해제** |

기본값: `TTL=900s(15분)`, 하트비트 `300s(5분)`마다, 획득 재시도 `5회 × 10초` 후 중단.

## 4. 구현 (bash, 그대로 사용)

```bash
VAULT="/Users/woodncarpenter/test-vault"
LOCK="$VAULT/.vault.lock"
TTL=900

# ── 획득: 성공 0 / 잠김(활성) 1 ─────────────────────────────
lock_acquire() {  # 인자: agent, session, op, note
  for i in 1 2 3 4 5; do
    if mkdir "$LOCK" 2>/dev/null; then
      now=$(date +%s)
      cat > "$LOCK/owner" <<EOF
agent: $1
session: $2
op: $3
pid: $$
host: $(hostname)
acquired_at: $(date "+%Y-%m-%dT%H:%M:%S+09:00")
heartbeat_at: $(date "+%Y-%m-%dT%H:%M:%S+09:00")
heartbeat_epoch: $now
ttl_seconds: $TTL
note: "$4"
EOF
      return 0
    fi
    # 잠김 — stale인지 확인
    hb=$(grep -m1 '^heartbeat_epoch:' "$LOCK/owner" 2>/dev/null | awk '{print $2}')
    now=$(date +%s)
    if [ -n "$hb" ] && [ $((now - hb)) -gt "$TTL" ]; then
      who=$(grep -m1 '^agent:' "$LOCK/owner" 2>/dev/null | cut -d' ' -f2-)
      echo "⚠️ stale 락 회수 (보유자: ${who:-unknown}, ${hb} 이후 무응답)"
      rm -rf "$LOCK"; continue        # 다음 루프에서 재획득
    fi
    echo "🔒 잠김 — 재시도 $i/5 (10초)"; sleep 10
  done
  echo "❌ 락 획득 실패: 다른 에이전트가 활성. 작업 중단."; return 1
}

# ── 하트비트: 긴 작업 중 주기적으로 호출 ──────────────────────
lock_heartbeat() {
  [ -f "$LOCK/owner" ] || return 1
  now=$(date +%s)
  sed -i.bak -E "s/^heartbeat_at:.*/heartbeat_at: $(date '+%Y-%m-%dT%H:%M:%S+09:00')/; s/^heartbeat_epoch:.*/heartbeat_epoch: $now/" "$LOCK/owner"
  rm -f "$LOCK/owner.bak"
}

# ── 해제: 작업 끝나면 반드시 ─────────────────────────────────
lock_release() { rm -rf "$LOCK"; }

# ── 상태 확인(읽기) ─────────────────────────────────────────
lock_status() {
  [ -d "$LOCK" ] && { echo "🔒 잠김:"; cat "$LOCK/owner"; } || echo "🔓 해제됨"
}
```

### 사용 패턴
```bash
if lock_acquire "Claude Code" "knowledge-$(date +%s)" "ingest" "옵시디언 영상 정제"; then
  trap 'lock_release' EXIT      # 크래시/종료 시 자동 해제 보장
  #  ... /ingest 작업, 중간중간 lock_heartbeat ...
  lock_release; trap - EXIT
fi
```
> `trap '... ' EXIT`로 **비정상 종료에도 해제**되게 한다(TTL은 그래도 못 푼 경우의 안전망).

## 5. 멀티 머신 주의
이 락은 **하나의 로컬 작업 사본** 안에서만 안전하다. 여러 기기가 같은 볼트를 동기화하며 동시에 쓰면 락 디렉터리 전파가 지연돼 보호가 깨진다. 이 경우는 락이 아니라 **사람 차원의 단일 작성자 규율**(한 기기만 편집)로 막는다. → `.vault.lock`을 **동기화 제외** 목록에 넣는다.

## 6. 통합 지점 (배선 완료)
§4의 함수는 **`scripts/vault-lock.sh`** 에 떨어져 있다. 소스(함수)·CLI(서브커맨드) 양쪽으로 쓸 수 있다.
```
bash scripts/vault-lock.sh acquire "<agent>" "<session>" "<op>" "<note>"   # 0=성공 1=실패
bash scripts/vault-lock.sh heartbeat | release | status
```
- ✅ **`/ingest`**: 대상 확정 후 노트 생성 전 `acquire`, 끝나면 `release`(통과 실패로 중단해도 해제).
- ✅ **`/lint --fix`**: 수정 전 `acquire`, 끝나면 `release`. (보고만 하는 기본 모드는 읽기 전용 → 락 불필요)
- `HERMES.md`·`CLAUDE.md`: 쓰기 작업 전 락 획득을 **의무**로 명시함.
- 동기화 ignore에 `.vault.lock/` 추가(클라우드 전파 지연으로 보호가 깨지지 않게).
