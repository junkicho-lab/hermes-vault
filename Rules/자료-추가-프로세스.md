# 자료를 볼트에 넣는 프로세스 (캡처 → ingest → 활용)

이 볼트에 새 자료를 넣을 때 따르는 실전 절차. 규칙의 *근거*는 [룰북](../CLAUDE.md)·[스킬](../.claude/skills/hermes-multi-agent-wiki/SKILL.md)에 있고, 여기는 *실행 방법*만 정리한다.

## 0. 큰 그림

```
외부 자료 ─① 캡처→ inbox/ (Raw) ─② /ingest→ canonical/ (Wiki) ─③ /query→ 활용
                                      └─ /lint (주기 정비)
```

**두 가지 원칙만 기억한다.**
- **캡처할 땐 거르지 않는다.** 좋다 싶으면 일단 inbox에 던진다. 가치 판단은 ②로 미룬다.
- **거르는 일은 ingest가 한다.** 필터를 통과한 것만 Wiki에 남는다. 통과 못 하면 inbox에 그대로 둔다.

---

## 1. 자료 유형별 캡처 방법

| 유형 | 방법 |
|------|------|
| **유튜브 영상** | 확장 프로그램(vidIQ 등)으로 스크립트 전체 복사 → inbox에 MD로 저장. `source_type: video`, `source_key`에 영상 URL |
| **웹 글·뉴스레터** | 본문 복사 → inbox MD. 또는 클로드에게 "이 내용을 inbox에 MD로 넣어줘". `source_type: web` |
| **PDF·책** | 핵심 텍스트 추출 → inbox MD. `source_type: pdf` 또는 `book` |
| **내 메모·아이디어** | 직접 inbox에 작성. `origin: internal`, `source_type: note` |

> 자동화: 유튜브 API/MCP, 메일, 노션 MCP를 연결하면 캡처를 자동화할 수 있다(클로드 코드에 연결).

---

## 2. inbox에 파일 만들기

- **위치**: `inbox/`
- **파일명**: 내용을 한 줄로 요약한 한글 케밥 (`옵시디언-클로드-결합.md`)
- **본문**: 가공·요약하지 말고 **날것 그대로** 붙여 넣는다.
- **frontmatter**: 아래 양식을 맨 위에 둔다.

**외부 자료 (origin=external)**
```yaml
---
source_type: web | video | book | newsletter | pdf
source_key: https://...        # 원본 URL
captured_at: 2026-06-09T10:40:00+09:00
title: 원본 제목
trust_tier: 2                  # 1=직접경험 2=외부·2차 3=미검증
origin: external
---
```

**내 메모 (origin=internal)**
```yaml
---
source_type: note
source_key: internal://주제-식별자
captured_at: 2026-06-09T10:40:00+09:00
title: 메모 제목
trust_tier: 1
origin: internal
---
```
> `captured_at`은 터미널 `date "+%Y-%m-%dT%H:%M:%S+09:00"`로 얻는다.

## 3. `/ingest` 실행

```
/ingest 옵시디언-클로드-결합.md      # 특정 파일
/ingest                            # 인자 없이 → 미정제 파일 목록에서 선택
```

ingest가 하는 일:
1. **필터 4관문** 판정 — 재사용성 · 영역 적합 · 비중복 · 변환 가능. (못 넘으면 노트 안 만들고 보고, 원본은 inbox 보존)
2. **원자 분해** — 통찰이 여러 개면 노트를 나눈다(한 노트 = 한 개념).
3. **영역 배정** — `전략-인사이트` / `콘텐츠-아이디어` / `방법론-노하우` / `학습-이론` 중 하나.
4. **중립 재서술** — 원저자 주장·구조·반례·출처만 살리고 군더더기·추론은 버린다.
5. `canonical/`에 저장 + 관련 노트 `[[링크]]` 연결.

---

## 4. 결과 확인 (내가 하는 일) ★

ingest는 **확인만 내 몫**이다. canonical 노트를 열어 점검한다.
- [ ] **영역 배정**이 맞는가?
- [ ] **제목**이 원저자 주장/개념인가(에이전트 해석 ❌)?
- [ ] 본문에 **에이전트의 의심·평가가 섞이지 않았는가**(중립 재서술인가)?
- [ ] `## 내 생각 / 질문` **여백을 내가 채운다** ← 반성·의심·적용 아이디어는 여기에. (에이전트는 비워 둠)
- [ ] `(제안)` 표시된 링크는 나중에 노트를 만들지 결정.

---

## 5. 활용과 정비

```
/query 정보 필터링은 어떻게 하지?     # canonical에서 근거를 찾아 답(읽기 전용)
/lint                              # 미정제 Raw·끊긴 링크·형식 위반 점검
/lint --fix                        # 안전한 자동 수정만 적용
```

> 입구(캡처)를 연 만큼 출구도 돌린다 — 주기적으로 `/ingest`로 inbox를 비우고 `/lint`로 점검하지 않으면 inbox가 무덤이 된다.

---

## 6. 자주 하는 실수

| 실수 | 바로잡기 |
|------|----------|
| inbox에서 미리 요약·편집 | 날것 그대로 넣고 정제는 ingest에 맡긴다 |
| 흥미롭다고 다 통과시킴 | 필터 4관문을 지킨다. 안 맞으면 inbox에 남긴다 |
| 한 노트에 여러 개념 | 원자성 — 나눠서 교차 링크 |
| `## 내 생각 / 질문`을 비워 두고 잊음 | 여기가 핵심. 내 사고를 채우는 자리다 |
| inbox만 쌓고 ingest 안 함 | 정기적으로 비운다 |
