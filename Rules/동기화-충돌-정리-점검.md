옵시디언 볼트에서 파일이 되살아나거나 `이름 1.md`·`(original)` 같은 중복본이 생길 때 따라가는 진단·정리 체크리스트. 나중에 검토용.

> 작성 계기: 스킬 폴더를 `.claude/skills/`로 옮긴 뒤 `Rules/`에 동일본이 되살아나고 `(original)`·`README 1.md` 충돌 사본이 생긴 사례(2026-06-09).

## 1. 무엇이 동기화 중인지 확정
- [ ] 볼트가 **iCloud "데스크탑 및 문서"**, Dropbox, OneDrive, 구글드라이브 폴더 *안*에 있나? (시스템 설정 → Apple 계정 → iCloud → iCloud Drive → "데스크탑 및 문서 폴더")
- [ ] 옵시디언 **Sync 플러그인**이 켜져 있나? (`.obsidian/sync.json` 존재 여부) — 이 볼트엔 없음
- [ ] `.obsidian/` 폴더 자체를 동기화하나? (워크스페이스 충돌의 주범)

## 2. 충돌 사본 전수 검색 (정리 대상 목록화)
```bash
# 폴더형
find . -type d \( -name '*(original)*' -o -name '*conflict*' \)
# 파일형
find . -type f \( -name '* [0-9].md' -o -name '*충돌*' \
   -o -iname '*conflict*' -o -name '*(1)*' \)
```
흔한 패턴: `이름 1.md`, `이름 (1).md`, `(original)`, `conflicted copy`, `충돌된 사본`

## 3. 어느 게 정본인지 판별 (지우기 전 필수)
- [ ] **수정시각**(`ls -lt`)과 **내용 diff**(`diff A B`)로 최신본 확인
- [ ] 정본 위치 합의: 이 볼트의 스킬 정본은 **`.claude/skills/`**, 잉여는 `Rules/` 쪽

## 4. 안전 정리 순서 (충돌 핑퐁 방지)
- [ ] **옵시디언 완전 종료** (열려 있으면 `workspace.json`·잠금 충돌 재발)
- [ ] 동기화 중이면 **일시 정지** → 정리 → 재개
- [ ] 정본만 남기고 충돌본 삭제 → **한쪽에서만** 정리(여러 기기 동시 편집 금지)
- [ ] 정리 직후 sync 한 사이클 **완료까지 대기** 후 결과 확인

## 5. 재발 방지
- [ ] **동기화 도구는 하나만** (iCloud+옵시디언Sync+git 중복 금지)
- [ ] 동기화 제외: `.DS_Store`, `.claude/`, `.obsidian/workspace.json`
- [ ] 여러 기기/세션 **동시 편집 금지**(한 곳 닫고 다른 곳 열기)
- [ ] git 사용 시 `.gitignore`에 `.DS_Store`, `**/workspace.json` 추가

## 6. 옵시디언 특화
- [ ] `.obsidian/workspace.json`은 기기마다 달라 충돌 단골 → **동기화 제외** 권장
- [ ] 옵시디언 휴지통(`.trash`) 경유 설정이면 삭제 후 복구 가능
