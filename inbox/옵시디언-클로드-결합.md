# 옵시디언과 클로드 코드의 결합

> 정보의 양은 줄이고 밀도는 높인다 — 내 사업에 진짜 필요한 정보만 추출하는 시스템

## 핵심 문제의식

- 정보를 **모으기만** 하고 정작 내 사업에는 적용되지 않는다.
- 책·강의·뉴스레터를 소비하지만 "어디서부터 무엇을 적용해야 할지" 막막하다.
- **도구를 잘 쓰는 것**과 **도구를 사업에 잘 활용하는 것**은 전혀 다르다.
- 진짜 문제는 정보의 양이 아니라 **나만의 필터가 없었던 것**.

## 옵시디언이란?

- **마크다운(.md) 파일 기반**의 노트 도구.
- 노션처럼 클라우드에 저장되지 않고, **내 컴퓨터 폴더에 MD 파일이 그대로 쌓인다.**
- 옵시디언이 말하는 **볼트(Vault)** = 내 컴퓨터의 폴더 하나.
- 노트끼리 `[[대괄호]]`로 연결되어, 자료들이 **시각적 그래프**로 연결된다 → "내 두 번째 뇌".

## Raw / Wiki 분리 개념 (LLM Wiki 패턴)

전 OpenAI·테슬라 AI 책임자 **안드레 카파시(Andrej Karpathy)** 가 공개한 **LLM Wiki 패턴**에서 출발.

> 핵심 비유: **옵시디언 = IDE, LLM(클로드 코드) = 프로그래머, 위키 = 코드베이스**

### 도서관 비유

| 역할 | 비유 |
|------|------|
| 옵시디언 | 도서관 건물 |
| 클로드 코드 (LLM) | 사서 |
| 위키 | 정리된 책꽂이 |
| 나 | 책을 사오는 사람 |

→ 책(정보)을 입구에 두면, 사서(클로드)가 읽고 **나에게 필요한지 판단** → 통과하면 알맞은 책꽂이(위키)에 분류·연결 → 나는 **확인만** 한다. 당장 쓸모없는 건 창고에 보관.

### Raw vs Wiki

- **Raw**: 외부에서 들어온 **날것의 정보** (유튜브 영상, 강의, 뉴스레터, 책 발췌, PDF 등).
- **Wiki**: **내 사업에 꼭 필요한 내용만** 들어가는 곳. 내 기준에 맞지 않으면 들어올 수 없다.
- 이 필터링이 없으면 정보가 머리 위를 둥둥 떠다니고, 무엇이 진짜 필요한지 구분되지 않는다.

## 세팅 방법

### 1. 클로드 코드 준비
- 클로드 코드 = **터미널에서 작동하는 AI 작업 도구** (내 컴퓨터 폴더에 직접 접근).
- 작업할 폴더를 연다 (없으면 새로 생성, 예: `옵시디언 비즈니스`).

### 2. 옵시디언 설치 & 연결
- 옵시디언 설치 후 **"보관함 폴더 열기"** → 클로드 코드와 같은 폴더 선택.
- 이제 사서(클로드)와 이용자(나)가 **같은 도서관을 동시에** 보게 된다.

### 3. Raw / Wiki 폴더 + 슬래시 커맨드 생성
- 카파시의 LLM Wiki 가이드(깃허브) 원본 + **사업 맞춤 프롬프트**를 순서대로 입력.
- 클로드가 **2가지 질문** 인터뷰:
  1. 본인의 사업 활동은 무엇인가?
  2. 그 일에서 어떤 정보·자료를 자주 다루는가?
- 답변을 바탕으로 도메인·영역·키워드·결정 트리·압축 룰을 **자동 생성** → 검토 후 승인.

### 4. 새로고침
- 슬래시 커맨드는 직접 만든 것이라 바로 검색되지 않음.
- `Cmd + Shift + P` → **"개발자 창 다시 로드"** 로 새로고침하면 적용됨.

> 전체 과정: 두 번 질문 → 자동 생성 → 새로고침. 대기 시간 포함 약 10분.

## 슬래시 커맨드 3종 (운영 체계)

| 명령어 | 의미 | 역할 |
|--------|------|------|
| `/ingest` | 섭취하다 | Raw 폴더의 새 자료를 위키에 알맞게 분류·정제·박제 |
| `/query` | 질문·조회 | 위키에 쌓인 내용 중 필요한 정보를 큐레이션 (읽기 전용) |
| `/lint` | 문법·구조 검사 | 쌓인 데이터의 오류·미처리 자료를 자동 진단 |

- **위키는 정보가 쌓일수록 가치가 폭발한다.** 1주일치는 별것 아니지만 1·3·6개월 누적되면 강력한 자산.
- `/lint`는 초기엔 쓸 일이 적지만, 시간이 쌓이면 **필수**.

## 실제 운영 흐름 (예시: 유튜브 영상)

1. 확장 프로그램(예: **vidIQ**)으로 영상 스크립트 전체 복사.
2. 클로드에게 "이 대본을 Raw 폴더에 MD 파일로 넣어줘" → Raw에 저장.
3. `/ingest` 실행 → 룰북 기준으로 적합한 영역(예: 시스템/인사이트)에 자동 배정.
4. 결과: **2,000줄 → 128줄로 압축**, 크로스 링크 5개·직접 인용 3개로 정리되어 위키에 저장.

→ 30분 영상의 핵심을 **5분 만에** 내게 필요한 형식으로 파악 가능.

> 이전: 100개 저장 → 30개만 적용.
> 지금: 100개 중 50~70개 이상을 저장·적용 가능.

## 확장 (MCP 연동)

- 유튜브 API/MCP → 매주 올라오는 영상 자동 처리.
- 이메일, 노션 MCP 등도 연결 → 직접 조사·정리하는 시간 절약.
- 실제로 노션·이메일·유튜브를 모두 연결해 운영 중.

## 진짜 메시지

> 로우·위키·슬래시 명령어는 **도구일 뿐**. 진짜 핵심은 따로 있다.

- 새로운 AI 도구를 **쫓아다니는 것**이 본질이 아니다. (2~3년간 쫓아다닌 결과: 쓸 줄 아는 도구만 늘고 사업에 도움된 건 몇 개 안 됨)
- **AI 시대의 본질** = 어떤 도구가 *내 사업을 돌아가게* 하고, *반복 업무를 줄여주고*, *나를 편안하게* 만드는지 찾는 것.
- 핵심은 **정보를 자기 원칙으로 변환하는 시스템**을 만드는 것. 도구 정리가 정답이 아니다.
- **위치가 자유가 아니라, 시스템이 자유다.** 그 시스템 위에 본인의 원칙이 박혀 있으면 어디서든 가능하다.
