# 참조 — Hermes식 멀티 에이전트 위키 (상세 명세)

SKILL.md의 보조 문서. 스키마·필터·영역·작성 규칙·커맨드 명세 전문을 담는다. 필요할 때만 펼쳐 읽는다(progressive disclosure).

## A. Frontmatter 스키마

**inbox (Raw)**
```yaml
---
source_type: web | video | book | newsletter | pdf | note
source_key: (원본 URL 또는 식별자)
captured_at: YYYY-MM-DDThh:mm:ss+09:00
title: (제목)
trust_tier: 1(직접경험) | 2(외부·2차) | 3(미검증)
origin: external | internal
---
```

**canonical (Wiki)**
```yaml
---
produced_by_task: "어떤 작업으로 만들어졌는지"
agent: <에이전트명>
model: anthropic/claude-opus-4.8
session: knowledge-<슬러그>
area: 전략-인사이트 | 콘텐츠-아이디어 | 방법론-노하우 | 학습-이론
aliases: ["<띄어쓰기 링크용 별칭>"]   # 파일명이 케밥일 때만
source: inbox/<원본파일>.md
captured_at: (원본 승계)
source_type: (원본 승계)
trust_tier: (원본 승계)
origin: (원본 승계)
produced_at: YYYY-MM-DDThh:mm:ss+09:00
---
```
승계 필드(`captured_at`·`source_type`·`trust_tier`·`origin`)는 원본 inbox에서 그대로 가져온다. `produced_at`은 생성 시각.

## B. 필터 기준 (Raw → Wiki 통과 규칙)

다음을 **모두** 만족할 때만 통과:
- **재사용성**: 한 달·6개월 뒤에도 다시 쓸 압축된 통찰인가? (일회성 뉴스·단순 사실 ❌)
- **영역 적합**: 4개 영역 중 하나에 분명히 들어가는가?
- **비중복**: 기존 canonical의 단순 반복이 아닌가? (보강이면 기존 노트에 `[[링크]]`)
- **변환 가능**: 단순 정보가 아니라 원칙/통찰로 바꿀 여지가 있는가?

통과 못 하면 canonical을 만들지 말고 보고한다(원본은 inbox에 보존).

## C. 영역(Area) 분류 + 결정 트리

| area | 들어오는 것 | 핵심 질문 |
|------|-------------|-----------|
| `전략-인사이트` | 사고/사업 모델, 의사결정 원칙, 멘탈 모델 | "어떻게 판단할 것인가?" |
| `콘텐츠-아이디어` | 소재, 스크립트, 카피, 브랜딩 | "무엇을 만들 것인가?" |
| `방법론-노하우` | 워크플로, 도구 사용법, 템플릿, 검증법 | "어떻게 실행할 것인가?" |
| `학습-이론` | 책·강의·논문 요약, 개념, 프레임워크 | "무엇을 이해할 것인가?" |

**결정 트리**: 실행 절차/팁 → `방법론-노하우`. 판단·의사결정 기준 → `전략-인사이트`. 만들 거리 → `콘텐츠-아이디어`. 순수 개념·이해 → `학습-이론`. 둘 이상 걸치면 **가장 강한 하나**로 배정, 나머지는 `[[링크]]`.

## D. canonical 작성 규칙 — [내 노트 기준] (origin=external)

형식 원본: `templates/외부자료-원자노트-템플릿.md`

- **본문 = 충실·중립 재서술만**. 1인칭 반성·의심·평가를 본문에 쓰지 않는다. 원문 복붙도 ❌.
- **반성 layer 분리**: 끝의 `## 내 생각 / 질문`은 빈 `-` 한 줄로 비운다. 사용자 여백. ← 대필 차단
- **원자성**: 한 노트 = 한 개념, 한 화면. 여러 개념이면 나눠 교차 링크.
- **제목**: 개념 명사구 또는 *원저자 주장*을 옮긴 한 문장. 에이전트 해석 ❌.
- **형식**: 산문 위주. 불릿은 꼭 필요한 열거만.
- **살릴 것**: 원저자 핵심 주장·구조·반례·출처.
- **버릴 것**: 서론 군더더기, 마케팅 어투, 중복, 원문에 없는 추론·의심.
- **링크**: 핵심 개념에 `[[링크]]`. 없는 개념은 `[[개념]] (제안)`. 파일명이 케밥인데 링크가 띄어쓰기면 대상 노트에 `aliases` 추가 또는 `[[파일명|표시]]` 형태 사용.
- **출처 불확실**: 자막·2차 자료면 "원 자료 확인 필요"를 본문에 중립적으로 명시.

## E. 슬래시 커맨드 명세

번들된 정의 파일: `templates/ingest.command.md`, `templates/query.command.md`, `templates/lint.command.md`
(설치 시 `.claude/commands/`에 `ingest.md`·`query.md`·`lint.md`로 배치)

### `/ingest [파일]` — Raw → canonical (쓰기)
1. 인자 없으면 inbox에서 미정제 파일(어떤 canonical도 `source:`로 미참조) 나열 후 선택.
2. 대상 읽기 → **필터 판정**(통과 못 하면 노트 생성 없이 보고).
3. 원자 분해 + 영역 배정(여러 통찰이면 노트 분리).
4. [내 노트 기준] 중립 재서술 + 말미 여백 비움 + `[[링크]]`(없으면 `(제안)`).
5. `canonical/<제목>.md` 저장(한글 케밥). 원본 inbox 보존.
6. 보고: 통과 여부·생성 노트·압축률·크로스링크 수.

### `/query <질문>` — 큐레이션 (읽기 전용)
canonical(필요시 inbox)을 훑어 질문에 답하고 `[[노트]]`로 인용. 파일을 쓰지 않는다. 근거 없으면 "관련 노트 없음" + 어떤 자료를 ingest하면 채워지는지 제안.

### `/lint [--fix]` — 건강 진단
점검: 미정제 Raw · frontmatter 결손 · placeholder 잔존 · 끊긴 위키링크 · 고아 노트 · 형식 위반(말미 섹션·원자성). 기본은 보고. `--fix`는 **안전 자동 수정만**(누락 필드·명백한 표시), 의미 판단(미정제 ingest·영역 배정·고아 연결)은 제안만.

## F. 설치 (새 볼트에 이식)

1. 볼트(폴더) 생성, 옵시디언과 클로드 코드를 같은 폴더에 연결.
2. `inbox/`·`staging/`·`canonical/` 폴더 생성.
3. `templates/외부자료-원자노트-템플릿.md` → 볼트의 `output-template/`로 복사.
4. `templates/*.command.md` 3종 → `.claude/commands/`에 `ingest.md`·`query.md`·`lint.md`로 복사.
5. `CLAUDE.md` 룰북 작성: 폴더구조·필터기준·영역·작성규칙. 인터뷰 2문항(무슨 일을 하나 / 어떤 자료를 다루나)으로 필터·영역을 자기 도메인에 맞게 채운다.
6. 자료를 `inbox`에 캡처 → `/ingest` → `/query` → 주기적 `/lint`.
