# Hermes식 멀티 에이전트 위키

외부 정보를 **모으는** 게 아니라 **내 원칙으로 변환**하는 지식관리 시스템. 카파시의 LLM Wiki 패턴(옵시디언=IDE, LLM=프로그래머, 위키=코드베이스)을 1인 PKM에 맞게 변형한 것이다.

## 언제 쓰나
- 옵시디언+클로드 코드로 "두 번째 뇌"/노트 볼트를 세우거나 운영할 때
- inbox(Raw) 자료를 정제(ingest)해 canonical(Wiki) 원자 노트로 만들 때
- 원자 노트 작성 규칙·필터 기준·영역 분류·슬래시 커맨드 운영이 필요할 때

## 멘탈 모델 (도서관)
옵시디언=도서관 · LLM(클로드 코드)=사서 · 위키=책꽂이 · 나=책 사오는 사람.
책(정보)을 입구에 두면 사서가 읽고 *나에게 필요한지 판단* → 통과분만 분류·연결, 쓸모없는 건 창고. **나는 확인만** 한다.

## 폴더 구조
| 폴더 | 역할 |
|------|------|
| `inbox/` | **Raw** — 외부 날것 정보 (캡처, 판단은 미룸) |
| `staging/` | 작업 중 임시 산출물 |
| `canonical/` | **Wiki** — 정제된 원자 노트 |
| `output-template/` | 작성 형식 원본 |
| `.claude/commands/` | `/ingest`·`/query`·`/lint` |
| `CLAUDE.md` | 룰북(필터·영역·규칙의 실제 기준) |

## 워크플로
```
외부 자료 ─캡처→ inbox(Raw) ─/ingest→ canonical(Wiki) ─/query→ 큐레이션
                                  └─ /lint (정비)
```
1. **캡처** — 자료를 `inbox/`에 MD로 던진다(가공·판단 없이). frontmatter는 inbox 스키마.
2. **`/ingest`** — 필터 통과분만 중립 재서술된 원자 노트로 정제, 4개 영역 중 하나 배정, `[[링크]]` 연결.
3. **`/query`** — canonical을 큐레이션해 활용(읽기 전용).
4. **`/lint`** — 미정제 Raw·끊긴 링크·형식 위반 주기 점검.

## 절대 규칙 (NON-NEGOTIABLE)
1. **필터가 먼저다.** 통과 못 한 자료는 canonical에 만들지 않는다(원본은 inbox 보존). 필터가 약해지면 두 번째 뇌가 잡동사니로 되돌아간다.
2. **중립 재서술만.** origin=external 노트 본문에 에이전트의 1인칭 반성·의심·평가를 쓰지 않는다. 원저자 주장·구조·반례·출처는 살리고, 원문에 없는 추론은 버린다.
3. **여백은 비운다.** 모든 canonical은 `## 내 생각 / 질문` 섹션을 빈 `-` 한 줄로 끝낸다. 그 칸은 사용자가 채운다. ← 대필 차단
4. **원자성.** 한 노트 = 한 개념. 여러 개념이면 나눠 교차 링크.
